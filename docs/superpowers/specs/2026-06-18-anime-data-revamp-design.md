# Anime 資料重整設計（公平選片 + 台灣繁體譯名）

日期：2026-06-18

## 背景與問題

`anime-data.ts` 目前的動畫清單來自 `init-data.ts`，是從 **bgm.tv（中國 Bangumi）** 依 `trends` 爬取的。
這導致兩個問題：

1. **選片偏中國宅圈口味**，使用者感覺「參雜個人/地區喜好」、不夠公平。
2. **譯名全是簡體 + 中國大陸譯名**（如「死亡笔记」），台灣使用者看不懂。

此外，`src/app.tsx` 寫死 `slice(0, 12)`（三處），實際只顯示每年前 12 部。

## 目標

把選片標準換成「全球客觀人氣」，並全面改用台灣繁體官方譯名，消除個人/地區偏好。

## 決策（已與使用者確認）

| 項目 | 決定 |
|---|---|
| 選片標準 | MyAnimeList 人氣（`members` 收藏/觀看人數），每年取前 N 名 |
| 資料來源 | Jikan API（MAL 公開 REST API） |
| 每年數量 | 固定 15 部 |
| 範圍 | 2006–2025，共 20 年 × 15 = 300 部 |
| 中文譯名 | 台灣官方繁體譯名；無正式代理者用台灣宅圈通用繁體譯名 |
| 英/日名 | 取 MAL 官方英文名與日文原名 |
| `score` 欄位 | 保留，填入 MAL 評分（不顯示，僅參考） |

## 選片規則細節

- 對每個年度，查詢「該年度首播（air start year）的 TV 動畫」。
- 依 `members` 由高到低排序，取前 15 名。
- 純依人數，不做人工挑選 → 結果可重現、可驗證。
- 陣列順序 = 人氣排序（app 會依陣列順序顯示）。

## 執行階段

### 階段一：抓取與選片驗證
- 寫抓取腳本（Jikan API），產出 `anime-raw.json`：每年前 15 名的
  `{ titleEn, titleJa, members, score }`。
- 交給使用者確認「選片是否公平」後再進入階段二。

### 階段二：譯名校對與產出
- 在原始清單上補台灣繁體 `titleZh`。
- 產出最終 `anime-data.ts`。
- **另外產出一份「沒有台灣官方譯名、以宅圈通用譯名暫代」的清單給使用者過目。**

## 程式改動

- 重寫 `anime-data.ts` 的 `data`（300 部）。
- `src/app.tsx` 三處 `slice(0, 12)` → `slice(0, 15)`（行 37、131、221）。
- `src/i18n.ts` 頁尾資料來源文字 bgm.tv → MyAnimeList（zh / en / ja 三語）。

## 不做（YAGNI）

- 不改畫面排版、不新增評分顯示、不動圖片匯出邏輯。
- 不做不相關的重構。
